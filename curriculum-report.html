VSCode & Claude Code
첫 걸음 +
커맨드 개념 이해
00:20 – 00:30

Claude Code 기본 사용법을 익히고, 슬래시 커맨드의 개념과 구조를 이해

  • VSCode 화면 구성 이해 — 탐색기, 편집창, Claude Code 채팅 패널
  • 첫 대화 테스트 — 간단한 질문으로 응답 흐름 확인
  • 슬래시 커맨드란? — .claude/commands/ 폴더 안의 .md 파일이 나만의 명령어가 되는 구조
  • 커맨드 파일 작성법 — 프롬프트를 파일로 저장하면 /파일명 으로 언제든 재실행
핵심 개념
  • 오늘 실습 2~5에서 만드는 에이전트는 모두 커맨드 파일로 저장됨
  • 교육 후 VSCode를 열고 슬래시(/) 만 치면 내가 만든 에이전트 목록이 뜸
3
실습 1
바이브코딩
— 말로 만드는 웹사이트
00:30 – 00:45 ★☆☆ 난이도 하

코드 한 줄 없이 자연어만으로 부서 소개 웹페이지를 만들고 실시간 수정

  • 바이브코딩 개념 소개 — "말하면 코드가 된다"는 경험 체득
  • 지시 예시: "우리 부서 소개 페이지 만들어 줘. 오렌지 컬러, 카드형 레이아웃으로"
  • 결과 확인 → 수정 요청 반복 — 색상·문구·레이아웃 즉시 반영 체험
  • 완성된 HTML 파일 저장 및 브라우저 미리보기
핵심 학습 포인트
  • 자연어 지시만으로도 실무 산출물이 만들어진다는 자신감 획득
  • 수정 → 피드백 → 재수정의 대화형 개발 루프 이해
4
실습 2
사장단 회의
준비 에이전트
00:45 – 01:00 ★★☆ 난이도 중

보고서를 붙여넣으면 회의 준비 자료를 즉시 뽑아주는 커맨드 파일 작성

  • 커맨드 파일 설계 — 역할 지정(경영 회의 준비 전문가), 입력 변수($ARGUMENTS) 구조 이해
  • 출력 항목 정의 — ① 핵심 3줄 요약 ② 예상 질문 5가지 ③ 리스크·취약 포인트
  • Claude Code에게 커맨드 파일 내용을 말로 지시해서 자동 생성
  • 실제 보고서로 /회의준비 호출 — 결과 확인 및 프롬프트 다듬기
📄
완성 산출물: 재사용 커맨드 파일 .claude/commands/회의준비.md → /회의준비 [보고서 텍스트]
5
실습 3
경영 의사결정
지원 에이전트
01:00 – 01:15 ★★☆ 난이도 중

상황을 입력하면 선택지 분석 → 권고안 → 반론(악마의 변호인)까지 자동 제공하는 커맨드 파일 작성

  • 멀티 페르소나 설계 — 권고자(찬성) + 비판자(반론) 두 역할을 하나의 커맨드에 정의
  • 출력 항목 정의 — ① 선택지 열거 및 장단점 ② 권고안 + 근거 ③ 악마의 변호인 반론
  • Claude Code에게 커맨드 파일 내용을 말로 지시해서 자동 생성
  • 실제 경영 상황으로 /의사결정 호출 — 결과 확인 및 기준 가중치 조정
📄
완성 산출물: 재사용 커맨드 파일 .claude/commands/의사결정.md → /의사결정 [상황 설명]
6
실습 4
회의록 → 담당자
자동 알림 에이전트
01:15 – 01:38 ★★☆ 난이도 중

회의록을 붙여넣으면 담당자별 할일 목록 + 개인화 이메일 초안을 자동 생성하는 커맨드 파일 작성

  • 파싱 로직 설계 — 이름·액션 아이템·기한 추출 + 긴급도(High/Mid/Low) 자동 분류 방식 정의
  • 출력 항목 정의 — ① 담당자별 할일 표 ② 개인화 이메일 초안 (담당자별 각 1건)
  • Claude Code에게 커맨드 파일 내용을 말로 지시해서 자동 생성
  • 샘플 회의록으로 /회의록 호출 — 긴급도 기준 수정 후 재실행 체험
📄
완성 산출물: 재사용 커맨드 파일 .claude/commands/회의록.md → /회의록 [회의록 텍스트]

  • 출력 항목 정의 — ① 오늘 하루 핵심 브리핑 ② 담당자별 메시지 초안 ③ 오늘의 CEO 명언
